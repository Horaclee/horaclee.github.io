Moteur Custom

  • Tech Stack: C++, DirectX 12
  • Github URL: Project Link

L’objectif était de développer un moteur de jeu personnalisé en C++ en utilisant DirectX 12 comme API graphique. Le moteur devait gérer le rendu 3D, la gestion des ressources, la physique et d’autres fonctionnalités essentielles pour un moteur de jeu.

Le projet étant en groupe, chaque membre devait travailler sur un module spécifique (ex. : rendu, physique, audio, gestion de scènes, etc.).


Architecture du moteur

  • Core : Gestion de la boucle de jeu et des événements.
  • Rendu (Renderer) : Utilisation de DirectX 12 pour l’affichage 3D.
  • Gestion des ressources : Chargement des modèles, textures, shaders.
  • Scripting : Ajout d’un système de scripting pour faciliter le développement.
  • Interface utilisateur (UI) : Création d’un éditeur pour manipuler les objets de la scène.

    • DirectX 12 : Gestion du rendu

  • Création d’une instance DXGI pour interagir avec le GPU.
  • Création d’un Device pour communiquer avec DirectX 12.
  • Création d’un Swap Chain pour gérer l’affichage des images.
  • Allocation des Command Queues, Command Lists et Command Allocators pour envoyer les commandes au GPU.
  • Mise en place d’un Descripteur de mémoire pour gérer les ressources (textures, buffers).

    • Pipeline graphique

  • Vertex Shader → Transformation des sommets.
  • Pixel Shader → Calcul des couleurs et de l’éclairage.
  • Rasterization → Conversion des triangles en pixels.
  • Depth & Stencil Testing → Gestion de la profondeur et des masques.
  • Output Merger → Finalisation de l’image avant affichage.

    • Gestion de la scène et des objets

  • Entity Component System (ECS) pour une gestion flexible des objets.
  • Chargement des modèles 3D (obj, fbx) via Assimp.
